# Spacer

[Properties](#properties) | [Examples](#examples) | [Upgrading](#upgrading)

`Spacer` is a component that applies `layout` and `space` properties to its children. This can be useful in conjunction with components like `Box` or `Flex`, where you may want to specify `padding` and `margin` for all children without having to do so individually for each child.

This replaces the `Layout` functionality for `childVerticalSpacing` and similar properties.

Generally, `Spacer` should be used only when there is a need to specify precise spacing between items. In most such cases, `Grid` may be more semantically correct so it is recommended to investigate that option first and to use this component sparingly.

## Properties

`Spacer` can use all `layout` and `space` properties. Note that these will override the values set for the child's properties, if any.

## Upgrading

This component is a **breaking change** that replaces the [previous Layout component](./?path=/docs/layout--page)'s `childSpacing` properties. Use the following table as a guide to convert from the old properties to new `Spacer` properties. Refer to [systemtheme.js](https://github.com/orchestrated-io/orca-design-system/blob/master/packages/orcs-design-system/lib/systemtheme.js) for values.

For `childChildSpacing` properties, wrap `<Spacer>` directly around the items you want to target and specify the desired properties.

### `Spacer` properties

| **Layout**                      | **Spacer**         |
| ------------------------------- | ------------------ |
| `childVerticalSpacing`          | `my={3}` |
| `childVerticalSpacingHalf`      | `my={2}`           |
| `childVerticalSpacingQuarter`   | `my={1}`           |
| `childHorizontalSpacing`        | `mx={3}` |
| `childHorizontalSpacingHalf`    | `mx={2}`           |
| `childHorizontalSpacingQuarter` | `mx={1}`           |
